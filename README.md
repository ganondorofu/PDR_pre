# PDR (Pedestrian Dead Reckoning) Android App

Pedestrian Dead Reckoning（歩行者推測航法）を使用したAndroidアプリです。スマートフォンの内蔵センサー（加速度計、磁力計、ジャイロスコープ）を使用して、ユーザーの歩行経路をリアルタイムで追跡・表示します。

## 主な機能

- **リアルタイム歩行追跡**: 加速度センサーを使用したステップ検出
- **方位推定**: 磁力計とジャイロスコープによる正確な方向判定
- **動的歩幅推定**: 歩行パターンに基づく歩幅の自動調整
- **視覚的な経路表示**: 歩行経路をグラフィカルに表示
- **統計情報**: 歩数、移動距離、記録状態の表示

## 技術的特徴

### PDRアルゴリズム
- **改良されたステップ検出**: 動的閾値とピーク検出を組み合わせたアルゴリズム
- **方位フィルタリング**: 移動平均を使用した角度の平滑化
- **歩幅適応**: 加速度の分散に基づく歩幅の動的調整

### センサー統合
- 加速度計: ステップ検出と歩幅推定
- 磁力計: 方位角計算
- ジャイロスコープ: 回転検出（将来の拡張用）

## 必要な権限

- `ACCESS_FINE_LOCATION`: 位置情報（センサー補正用）
- `ACCESS_COARSE_LOCATION`: 位置情報（センサー補正用）
- `ACTIVITY_RECOGNITION`: 活動認識

## インストールと実行

1. Android Studio で本プロジェクトを開く
2. Android SDK 26以上の端末またはエミュレータを準備
3. アプリをビルドしてインストール
4. 必要な権限を許可
5. 「開始」ボタンを押して歩行追跡を開始

## 使用方法

1. **開始**: 「開始」ボタンをタップして記録を開始
2. **歩行**: スマートフォンを持って歩く（ポケットや手に持つ）
3. **経路確認**: 画面上でリアルタイムに経路が描画される
4. **停止**: 「停止」ボタンで記録を停止
5. **リセット**: 「リセット」ボタンで経路をクリア

## アーキテクチャ

- **MVVM パターン**: ViewModelを使用した状態管理
- **Jetpack Compose**: モダンなUI構築
- **コルーチン**: 非同期処理（将来の拡張用）

## 精度向上のヒント

1. **キャリブレーション**: アプリ開始前に8の字に端末を動かす
2. **持ち方**: 端末を一定の向きで保持する
3. **歩行スタイル**: 一定のリズムで歩く
4. **環境**: 磁気干渉の少ない場所で使用する

## 制限事項

- GPS不使用のため、絶対位置ではなく相対位置のみ
- 長時間使用すると累積誤差が発生する可能性
- 端末の向きや持ち方により精度が変わる
